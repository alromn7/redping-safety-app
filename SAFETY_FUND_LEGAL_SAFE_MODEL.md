## üîÑ REPAYMENT SYSTEM (Sustainability Mechanism)

### How It Works:

**Normal State:** $5/month subscription

**After Using Safety Fund:**
- Contribution automatically increases to $15/month
- Extra $10/month goes toward repaying fund share
- Returns to $5/month when balance paid off
- Journey continues during repayment

**Example:**
```
Rescue cost: $3,000
User's journey stage: Stage 3 (90/10 split)
Fund assists: $2,700
User pays: $300

Repayment:
- Balance added: $2,700
- Monthly contribution: $15 ($10 toward debt)
- Months to pay off: 270 months (22.5 years)
- OR user can pay lump sum anytime
```

**Legal Basis:** This is a pricing structure for service usage, NOT insurance premium calculation.

## üõ°Ô∏è EXTERNAL COVERAGE SYSTEM

### The Smart Approach (NOT Bill Splitting):

#### Scenario A: User HAS External Coverage

**When emergency occurs:**
1. User uses THEIR insurance/WorkCover/etc.
2. External coverage handles rescue costs
3. Safety Fund NOT activated
4. RedPing provides coordination platform only
5. Journey continues normally
6. Subscription stays at $5/month ‚úÖ

**Legal:** We're NOT involved in payment. User uses their existing coverage. We provide communication platform.

#### Scenario B: User Has External BUT Chooses Safety Fund

**Penalty System (Discouragement, NOT Denial):**
1. User CAN still use Safety Fund (rescue never blocked)
2. Penalty billing split applies: 40% fund / 60% user
3. Makes it MORE expensive than using their own coverage
4. Repayment system activates ($5 ‚Üí $15/month)

**Example:**
```
Rescue cost: $3,000
Has WorkCover but didn't use it
Penalty split: 40/60
Fund assists: $1,200
User pays: $1,800
Repayment balance: $1,200
Time to pay off: 120 months (10 years)

VS if they used WorkCover:
WorkCover pays: $3,000
User pays: $0
No repayment
Journey continues
```

**Legal:** This is pricing structure based on service usage choices, NOT insurance coverage.

#### Scenario C: User Has NO External Coverage

**Normal Journey-Based Support:**
1. User pays rescue bill
2. Safety Fund assists with community pooling
3. Journey stage determines billing split (80-100% fund share)
4. Repayment system activates
5. Returns to $5/month when paid off

**Legal:** Voluntary community mutual aid, NOT insurance payout.

## üîê RISK MITIGATION

### Catastrophic Event Protection:

**Scenario: Major Disaster (bushfire, flood)**
- Multiple users need rescue simultaneously
- Fund could be depleted in hours

**Solutions:**

1. **Reinsurance Partnership:**
   - Partner with actual insurer for catastrophic events
   - Fund covers first $500 per rescue
   - Reinsurer covers $500-$10,000
   - Costs ~$2/user/month but caps fund exposure

2. **Emergency Fund Reserve:**
   - Maintain 12-month claims reserve
   - Monthly deposits to reserve account
   - Only accessible for catastrophic events
   - Board approval required

3. **Disaster Mode:**
   - Automatically activated during major events
   - Prioritizes coordination over funding
   - Users acknowledge emergency conditions
   - Government/charity assistance coordination

4. **Fund Health Monitoring:**
   - Real-time fund balance tracking
   - Automatic alerts if balance drops
   - Slow down claim processing if needed
   - Transparent communication to community
